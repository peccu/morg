# /task — タスク管理スキル

`TASKS.md` と `.claude/rules/lessons.md` を管理するスキルです。

## 使い方

```
/task add          新しいタスクを追加する
/task update T001  既存タスクのステータス・内容を更新する
/task lesson       新しい Lesson を lessons.md に追加する
/task list         現在のタスク一覧を表示する
```

## タスク追加 (`/task add`)

1. `TASKS.md` を読み込む
2. 既存の最大タスク番号 + 1 で新しいID（T00N）を決める
3. ユーザーから以下を確認する（会話で聞く）:
   - タスク概要（1行）
   - 優先度（高/中/低）
   - 調査内容・背景（わかっている範囲で）
   - 対応方針（未定ならTBD）
   - 確認方法
4. テンプレートに従ってセクションを追記する
5. ステータスは `🔍 調査中` または `📋 対応予定` を選ぶ

## タスク更新 (`/task update TXXX`)

1. `TASKS.md` を読み込み、対象タスクを見つける
2. 変更内容をユーザーに確認する:
   - ステータス変更
   - 調査内容・対応方針・確認方法の追記
3. 該当セクションのみ編集する

## Lesson 追加 (`/task lesson`)

1. `.claude/rules/lessons.md` を読み込む
2. 最大番号 + 1 で新しいID（L00N）を決める
3. ユーザーから以下を確認する:
   - 何が発生したか（発生状況）
   - 症状（エラーメッセージ等）
   - 対策（次回どうすべきか、コード例があれば含める）
4. 以下のテンプレートで追記する:

```markdown
## L00N · タイトル

**発生**: いつ・どの作業中に

**症状**
\```
エラーメッセージや症状
\```

**対策**: 具体的な対処法。コード例があれば示す。
```

## 注意事項

- `TASKS.md` の凡例・構造を崩さない
- lessons.md は**毎セッション自動ロード**されるため、冗長な内容は書かない（簡潔に）
- タスクIDは欠番を作らず連番にする
